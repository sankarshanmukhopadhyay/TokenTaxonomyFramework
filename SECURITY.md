# Security Policy

This repository primarily publishes standards documentation and machine-readable artifacts. Security issues may still arise where examples, schemas, proto definitions, generated documentation, or links could mislead implementers or create unsafe implementation guidance.

## Report a security issue

Please report suspected security issues through the repository maintainers or the responsible InterWork Alliance channel. Do not disclose exploitable issues publicly until maintainers have had an opportunity to assess and respond.

## Security-relevant changes

Security-relevant pull requests should describe:

- affected artifact or documentation scope;
- possible implementation impact;
- whether existing consumers need migration guidance;
- validation evidence produced by repository scripts;
- whether any artifact should be deprecated or replaced.

## Non-security limitations

The repository validation scripts check repository integrity. They do not certify that an ecological methodology, registry process, token design, or implementation is economically, legally, scientifically, or operationally correct.
