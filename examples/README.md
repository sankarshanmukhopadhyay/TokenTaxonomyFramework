# Examples

This directory contains lightweight examples that help adopters understand how to use the Token Taxonomy Framework repository as an implementation and assurance surface.

| Example | Purpose |
|---|---|
| [minimal-token-artifact](minimal-token-artifact/) | Shows the minimum documentation pattern for a small token artifact contribution. |
| [dmrv-extension-set](dmrv-extension-set/) | Shows the expected package shape for a dMRV extension-set contribution. |
| [validation-report](validation-report/) | Shows how validation evidence can be captured for a pull request or release. |
| [regulated-tokenization](regulated-tokenization/) | Shows v1.3.0 governance metadata for permissioned RWA, tokenized bond, stable-value, dMRV claim, and verifiable document patterns. |
