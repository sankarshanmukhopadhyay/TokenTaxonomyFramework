# Extension Set Checklist

Use this checklist when creating or reviewing a dMRV extension set.

## Authority and scope

- [ ] Methodology, quality standard, registry, or governing process is identified.
- [ ] Extension-set owner or contributor is identified.
- [ ] Scope boundaries are documented.
- [ ] Known limitations are documented.

## Deployment package

- [ ] Extension set metadata is present.
- [ ] Entity extension templates are present.
- [ ] Formula templates are present.
- [ ] Variable templates are present.
- [ ] Message pairs are present.
- [ ] Proto definitions are included where needed.

## Instance package

- [ ] Claim sources are present.
- [ ] AIM fixed variables are present.
- [ ] Example bindings are internally consistent.

## Evidence and validation

- [ ] JSON validation passes.
- [ ] Internal link validation passes.
- [ ] Artifact structure check passes.
- [ ] Catalogs are regenerated and current.
- [ ] Human review assumptions are documented.
