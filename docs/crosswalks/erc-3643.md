# ERC-3643 Permissioned Token Crosswalk

ERC-3643 is an identity-aware token standard for permissioned tokens, especially regulated real-world assets. TTF can represent the same design intent in implementation-neutral terms.

| ERC-3643 concept | TTF mapping |
|---|---|
| Identity registry | `transferControls.eligibilityCredentialRequired` and identity evidence references. |
| Claim topics and trusted issuers | `authority`, `delegationScope`, and `evidence.requiredEvidence`. |
| Compliance module | `compliant`, `roles`, `transferable`, and policy references. |
| Transfer restrictions | `transferControls.permissioned`, jurisdictional restrictions, holding rules, and transfer preconditions. |
| Forced transfer, freeze, or recovery | Lifecycle controls, revocation reference, pause/suspension behavior, and audit log requirements. |

## Design Guidance

Do not model a regulated transfer as plain `transferable`. A permissioned token should state which policy or credential check gates transfer, who can update that policy, and what evidence is produced when a transfer is rejected or approved.
